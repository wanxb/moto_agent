# 用户画像 · 场景 · Jobs-to-be-Done

> 本文件补充 [`../PRD.md`](../PRD.md) §2 的目标用户，提供更细的画像与场景，用于指导功能优先级与交互设计。

---

## 1. 用户画像

### Persona A：「自用骑手」开发者本人（MVP 主用户）

| 维度 | 描述 |
|------|------|
| 身份 | 有技术背景的摩托车爱好者，能自行部署 Cloudflare/Telegram |
| 车辆 | 1 辆（Phase 2 可能 2–3 辆） |
| 使用频率 | 每周 1–3 次加油记录，偶尔查询 |
| 核心诉求 | 加油后 5 秒完成记录，不打断生活；长期积累数据看趋势 |
| 痛点 | 备忘录/表格记录繁琐、易漏、难统计 |
| 技术容忍度 | 高——能接受偶发延迟，但讨厌"还要学指令格式" |
| 设备 | 手机 Telegram 为主 |

### Persona B：「省心骑手」非技术用户（Phase 3 目标）

| 维度 | 描述 |
|------|------|
| 身份 | 普通通勤/玩乐骑手，不懂部署 |
| 车辆 | 1 辆 |
| 使用频率 | 每周 1–2 次 |
| 核心诉求 | 别让我学习；告诉我"这个月花了多少油钱""油耗正不正常" |
| 痛点 | 现有 App 太重、要注册、广告多 |
| 技术容忍度 | 低——任何报错/卡顿就流失 |
| 设备 | 手机，可能用微信多于 Telegram（Phase 4 多平台诉求来源） |

> MVP 只服务 Persona A。Persona B 是 Phase 3「多用户 + 可视化」的设计锚点，现在记录下来以免后续架构把路堵死。

## 2. 核心场景（User Stories）

格式：作为 **<角色>**，我想 **<行为>**，以便 **<价值>**。

### 记录类

- **S1（加油记录）** 作为骑手，我想在加油站直接发一句"加了 10 升 95 号花了 98，里程 12580"，以便 5 秒内完成记录而不用填表。
- **S2（模糊补全）** 作为骑手，当我只说"加油 300 块"时，我想让 Bot 主动问缺的信息，以便不必一次说全也能记上。
- **S3（纯里程）** 作为骑手，我想偶尔只记一下当前里程，以便后续区间油耗计算更准。
- **S4（多车切换｜Phase 2）** 作为有多辆车的骑手，我想说"小绿加油…"指定车辆，以便每辆车独立统计。

### 查询类

- **S5（本次油耗）** 作为骑手，我想在记录后立刻看到"本次油耗 3.13 L/100km"，以便即时感知车况。
- **S6（区间统计）** 作为骑手，我想问"最近 3 个月油耗怎么样"，以便了解趋势和花费。
- **S7（最近记录）** 作为骑手，我想问"上次什么时候加的油"，以便决定是否该加油了。

### 维护类（Phase 2）

- **S8（保养记录）** 作为骑手，我想记录"换机油 里程 13000"，以便追踪保养历史。
- **S9（保养提醒）** 作为骑手，我想在接近换机油里程时收到提醒，以便不错过保养。

### 体验类

- **S10（零学习）** 作为新用户，我想发 `/start` 就知道怎么用，以便立即上手。
- **S11（纠错｜Phase 2）** 作为骑手，当我记错了数字，我想说"上一条里程改成 12680"，以便修正数据。

## 3. Jobs-to-be-Done（JTBD）

| 当我…（情境） | 我想…（动机） | 以便…（期望结果） |
|--------------|--------------|------------------|
| 刚加完油站在油枪旁 | 立刻把数据记下来 | 不用回家补录、不会忘 |
| 月底看账 | 知道这个月油费多少、油耗如何 | 心里有数，发现异常 |
| 准备长途骑行前 | 知道车最近油耗是否正常 | 判断车况是否健康 |
| 车子感觉"喝油" | 对比历史油耗确认 | 及时发现潜在故障（Phase 4 异常检测） |
| 保养周期临近 | 被提醒该换机油/保险了 | 不漏保养、不超期 |

## 4. 反面用户（Non-Goals）

明确**不**为以下人群优化，避免范围蔓延：

- 车队/商用车队管理者（多车多司机、报销流程）——超出个人工具定位。
- 重度数据分析师（需要 BI、自定义报表导出 API）——Phase 3 Dashboard 提供基础图表即止。
- 需要离线使用的用户——本产品依赖网络与 LLM。

## 5. 设计原则（从画像推导）

1. **对话优先，命令兜底**：Persona A/B 都讨厌记格式，自然语言是第一交互；`/stats`、`/last` 等命令只作快捷兜底。
2. **即时反馈**：每次记录后立刻回算结果（S5），强化"记了有用"的正反馈。
3. **主动澄清而非报错**：信息不全时提问（S2），不要丢一句"格式错误"。
4. **数据是资产**：永不丢用户数据，schema 只增不删（见 [`../engineering/data-model.md`](../engineering/data-model.md)）。
5. **为 Persona B 预留但不提前实现**：多用户隔离的字段/接口提前在架构上留位，但 MVP 不做鉴权 UI。
