# 需求：维修保养记录

> 规格 002 · 状态：✔️ 已实现（待线上部署） · 阶段：Phase 2 · 优先级：P1
> 关联：[roadmap](../../product/roadmap.md) · [design.md](design.md) · [tasks.md](tasks.md) · 依赖 [spec 001 多车](../001-multi-vehicle/)

## 1. 问题陈述

骑手需要追踪保养历史（换机油、换轮胎、保险、刹车、链条等），以便知道"上次换机油是什么时候/多少公里""今年保险交了吗"。当前只记加油，保养无处可记（[S8](../../product/personas.md#维护类phase-2)）。维保记录也是**定时提醒（spec 003）**的数据来源——里程类提醒需要"上次保养里程"作基准。

## 2. 用户故事

- **US1** 作为骑手，我想记录一次保养（类型、里程、日期、费用），以便留存维保历史。
- **US2** 作为骑手，我想查询某辆车的保养历史，以便回顾做过什么。
- **US3** 作为骑手，我想问"上次换机油是什么时候"，以便判断是否该再保养。
- **US4** 作为多车骑手，我想保养记录绑定到具体车辆，以便分车追踪。

## 3. 范围

**In Scope**
- 记录保养：类型、日期、里程（可选）、费用（可选）、备注（可选），绑定车辆。
- 查询保养历史：全部 / 按类型 / 按车。
- 查询某类型最近一次保养。
- 复用 spec 001 的车辆解析（默认车 / 指定车 / 歧义反问 / 无车回退单车）。

**Out of Scope（本期不做）**
- 定时提醒 / 到期推送 —— spec 003。
- 保养记录的修改 / 删除 —— 与加油记录纠错一并放 P2。
- 预设保养周期模板（如"机油每 3000km"）—— spec 003 提醒时再设计。

## 4. 验收标准（Given / When / Then）

- **AC1（记录·默认车）** Given 有默认车，When 用户说"换机油 里程 13000 花了 80"，Then 记录一条 type=机油、odometer=13000、cost=80 的保养，归属默认车。
- **AC2（记录·指定车）** Given 多车，When 用户说"通勤车换轮胎 200 块"，Then 记录归属"通勤车"，type=轮胎、cost=200（odometer 可空）。
- **AC3（无里程保养）** When 用户说"交了保险 1200"，Then 记录 type=保险、cost=1200，odometer 为空（不强制）。
- **AC4（查询历史）** Given 有多条保养，When 用户问"小绿的保养记录"，Then 列出该车保养历史（按里程/日期倒序）。
- **AC5（查询某类型最近）** Given 有换机油记录，When 用户问"上次换机油"，Then 返回最近一次换机油的里程与日期。
- **AC6（歧义/无车）** 车辆解析行为与 spec 001 一致：多车无默认且未指明 → 反问；无任何车辆 → 记到单车（vehicle_id 空）不退化。

## 6. 依赖与假设

- 依赖：**spec 001 多车**（保养绑定 `vehicle_id`，复用 `resolveVehicle`）。
- 假设：保养类型以常见枚举为主但允许自由文本；里程/费用可缺省。

## 7. 开放问题

| 问题 | 影响 | 待决 |
|------|------|------|
| 保养类型是否固定枚举 | 影响"上次换机油"匹配准确性 | 采用"常见枚举 + 自由文本"，匹配用包含/相等 |
| 是否记录保养项的下次周期 | 提醒功能需要 | 留给 spec 003，本期不引入周期字段 |
