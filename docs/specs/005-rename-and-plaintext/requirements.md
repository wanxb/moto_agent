# 需求：车辆改名 + 纯文本输出

> 规格 005 · 状态：✔️ 已实现 · 阶段：Phase 2（体验优化） · 优先级：P2
> 关联：[design.md](design.md) · [tasks.md](tasks.md) · 依赖 [spec 001 多车](../001-multi-vehicle/)

两个独立的小改进，合并为一个轻量 spec。上线后用户真实反馈驱动。

## A. 车辆改名

### 问题
车辆建好后无法改名（如"小绿"想改成"大绿"）。历史记录通过 `vehicle_id` 关联，改名应自动作用于过去的所有记录显示。

### 用户故事
- **US-A1** 作为骑手，我想给已有车辆改名（"把小绿改名叫大绿"），以便修正/调整车名。
- **US-A2** 改名后，该车的**历史加油/保养/提醒**都应显示新名（因关联的是 id 而非名字）。

### 验收标准
- **AC-A1** Given 有车辆"小绿"，When 用户说"把小绿改名叫大绿"，Then 该车 `name` 更新为"大绿"。
- **AC-A2** Given 小绿名下有历史记录，When 改名为大绿后查询统计，Then 表头/回显显示"大绿"。
- **AC-A3** Given 已存在"通勤车"，When 想把"小绿"改名为"通勤车"，Then 提示重名、不改。
- **AC-A4** Given 没有"幽灵车"，When 要求改它的名，Then 提示没找到。

## B. 纯文本输出

### 问题
Bot 回复有时带 Markdown（`**粗体**`、`### 标题`、`- 列表`、`` `代码` ``），但 Telegram 默认按纯文本发送，这些符号**原样显示**，影响观感。来源是 LLM 自由回复（工具输出本身是纯文本+emoji）。

### 用户故事
- **US-B1** 作为骑手，我希望 bot 回复干净易读，不出现 `**`、`###`、反引号等 markdown 符号。

### 验收标准
- **AC-B1** When LLM 回复含 `**粗体**`，Then 用户看到的是"粗体"（去掉星号）。
- **AC-B2** When 回复含 `### 标题` / `` `代码` `` / `[文字](链接)`，Then 显示为纯文本（标题去 `#`、代码去反引号、链接保留文字）。
- **AC-B3** When 回复含 `- 项` 或 `* 项` 列表，Then 显示为 `• 项`。
- **AC-B4** 工具的既有输出（emoji、`─` 分隔线、`•`、`¥`、数字）**不被破坏**。

## 范围

**In Scope**：车辆改名工具；输出层 markdown 清洗 + prompt 约束。
**Out of Scope**：车辆删除/停用（软删 `is_active` 已预留，本期不做对话入口）；富文本/按钮等 Telegram 高级 UI。

## 依赖与假设
- 依赖 spec 001（车辆按 id 关联，改名零成本作用于历史）。
- 无数据库结构变更（改名是 UPDATE，清洗是代码）。
