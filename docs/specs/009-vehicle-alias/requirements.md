# 需求：车辆别名（简称）

> 规格 009 · 状态：✔️ 已实现（待部署） · 阶段：Phase 2（体验） · 优先级：P2
> 关联：[design.md](design.md) · [tasks.md](tasks.md) · 依赖 [spec 001 多车](../001-multi-vehicle/)

## 1. 问题陈述

车辆的正式名称可能较长（如"Honda NS125LA"），日常记录时用户想用简称（"小拉"）。当前系统只能精确匹配 `name`，用户要么每次打全名、要么一开始就用简称作全名（失去正式名称的含义）。需要支持**别名（简称）**：车辆有一个正式名称 + 一个可选简称，两种都能识别。

## 2. 用户故事

- **US1** 作为骑手，我想给车辆起一个简称（如"Honda NS125LA"简称"小拉"），以便日常用简称指代。
- **US2** 作为骑手，我说"小拉加了 5 升"时，系统能识别为 Honda NS125LA 并正确记录。
- **US3** 简称是唯一的（不能两辆车叫同一个简称）；可以为空（没有简称时只能用全名）。
- **US4（改名联动）** 改全名或改简称时，不影响彼此。

## 3. 验收标准

- **AC1（设简称）** Given 车辆"Honda NS125LA"，When 用户说"给 Honda NS125LA 起个简称叫小拉"，Then 该车 alias 设为"小拉"，提示确认。
- **AC2（通过简称指代）** Given 车辆别名"小拉"，When 用户说"小拉加了 5 升花 40 里程 35000"，Then 记录归属 Honda NS125LA。
- **AC3（通过全名指代不变）** When 用户说"Honda NS125LA 最近油耗"，Then 仍能识别为同一辆车。
- **AC4（简称唯一）** Given 已有一辆车别名"小拉"，When 再给另一辆车设别名"小拉"，Then 提示重名冲突。
- **AC5（列表显示简称）** 车辆列表显示全名 + 别名（如有）："Honda NS125LA（小拉）"。
- **AC6（改名联动）** 改全名或改简称互不影响。
- **AC7** 设置默认车可通过简称（"默认车设成小拉"）、取消默认车也通过简称。

## 4. 范围

**In Scope**：给车辆设别名；通过别名指代（resolve 匹配 name 或 alias）；列表显示别名。
**Out of Scope**：多个别名（一车多别名）——只有一个；别名自动补全/模糊匹配。
