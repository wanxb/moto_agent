# 需求：定时提醒

> 规格 003 · 状态：✔️ 已实现（待线上部署） · 阶段：Phase 2 · 优先级：P1
> 关联：[roadmap](../../product/roadmap.md) · [design.md](design.md) · [tasks.md](tasks.md) · 依赖 [spec 001 多车](../001-multi-vehicle/) + [spec 002 维保](../002-maintenance/)

## 1. 问题陈述

骑手容易错过保养与到期事项：机油到了里程该换、保险/年检到期。当前系统被动等用户来问，不会主动提醒（[S9](../../product/personas.md#维护类phase-2)）。本功能引入**主动推送**：到达里程阈值或日期时，Bot 主动发消息。

这是项目第一个**非 webhook 入口**——用 Cloudflare **Cron Triggers** 定时唤醒 Worker 扫描到期提醒。

## 2. 用户故事

- **US1（里程提醒）** 作为骑手，我想设"机油每 3000 公里提醒"，以便到里程时收到换机油提示。
- **US2（日期提醒）** 作为骑手，我想设"保险 2027-01-05 到期提醒"，以便到期前收到提示。
- **US3（查看）** 作为骑手，我想查看我设了哪些提醒。
- **US4（取消）** 作为骑手，我想取消某个提醒。
- **US5（主动推送）** 作为骑手，当提醒到达条件时，我想自动收到一条 Telegram 消息，无需主动查询。

## 3. 范围

**In Scope**
- 设置提醒：里程模式（绝对目标里程 或 "间隔 + 基准"）、日期模式。
- 列出 / 取消提醒。
- Cron 定时扫描到期提醒并主动推送（**一次性**：触发后标记完成）。
- 复用 spec 001 车辆解析；里程基准可取 spec 002"上次同类保养里程"。

**Out of Scope（本期不做）**
- 自动循环提醒（如每 3000km 自动重置）——本期一次性；用户可在换油后重设。下一迭代再做"换油即自动续期"。
- 提前 N 天/N 公里预警（只在到达阈值时提醒，不做提前量）——可作快速增强。
- 多用户独立推送目标——MVP 推送到 `ALLOWED_CHAT_ID`；表中预留 `chat_id` 列。

## 4. 验收标准（Given / When / Then）

- **AC1（设里程·间隔）** Given 默认车上次换机油里程 10000，When 用户说"机油每 3000 公里提醒"，Then 创建里程提醒，目标里程 = 13000。
- **AC2（设里程·绝对）** When 用户说"机油到 13000 公里提醒我"，Then 创建里程提醒，目标里程 = 13000。
- **AC3（设日期）** When 用户说"保险 2027-01-05 到期提醒"，Then 创建日期提醒 trigger_date=2027-01-05。
- **AC4（列出）** When 用户问"我设了哪些提醒"，Then 列出活跃提醒（类型、车、条件）。
- **AC5（取消）** Given 有机油提醒，When 用户说"取消机油提醒"，Then 该提醒置为完成、不再触发。
- **AC6（里程到达即触发）** Given 里程提醒目标 13000，When 最新记录里程 ≥ 13000，Then 扫描判定为到期。
- **AC7（日期到达即触发）** Given 日期提醒 2027-01-05，When 当天 ≥ 该日期，Then 扫描判定为到期。
- **AC8（触发后去重）** Given 一条提醒已触发推送，When 下次扫描，Then 不再重复推送（status=done）。
- **AC9（按车隔离）** 里程提醒只看其所属车辆的最新里程（不跨车）。

## 5. 交互示例

```
用户：机油每 3000 公里提醒我
Bot：🔔 已设置提醒（小绿）
     机油 · 里程达到 13,000 km 时提醒（上次换油 10,000 km + 3,000）

用户：保险 2027-01-05 到期提醒
Bot：🔔 已设置提醒（小绿）
     保险 · 2027-01-05 到期时提醒

用户：我设了哪些提醒
Bot：🔔 提醒列表（小绿）
     • 机油 · 13,000 km
     • 保险 · 2027-01-05

—— Cron 到期时主动推送 ——
Bot：🔔 保养提醒（小绿）
     该换机油了：当前 13,050 km ≥ 提醒里程 13,000 km
```

## 6. 依赖与假设

- 依赖：spec 001（车辆 + 解析）、spec 002（"上次同类保养里程"作里程基准）。
- 假设：单用户，推送目标为 `ALLOWED_CHAT_ID`；最新里程来自最近的加油/里程记录（用户不记录则里程提醒不会误触发）。
- Cron 频率：每日一次（保养提醒无需秒级精度）。

## 7. 开放问题

| 问题 | 影响 | 待决 |
|------|------|------|
| 里程提醒依赖用户记录里程 | 不记录则不触发 | 可接受；文档说明。日期提醒不受影响 |
| 一次性 vs 循环 | 换油后需重设 | 本期一次性；下一迭代做"记录同类保养时自动续期" |
| Cron 频率/时区 | 日期边界 | 每日 UTC 扫描；时区与 [data-model §6](../../engineering/data-model.md) 时区问题一并处理 |
