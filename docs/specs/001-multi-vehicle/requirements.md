# 需求：多车管理

> 规格 001 · 状态：✔️ 已实现（待线上部署 T12） · 阶段：Phase 2 · 优先级：P0
> 关联：[roadmap](../../product/roadmap.md) · [design.md](design.md) · [tasks.md](tasks.md)

## 1. 问题陈述

当前所有加油/里程记录共用一辆隐含的"车"，无法区分多辆摩托车。有多辆车的骑手（[Persona A](../../product/personas.md#persona-a自用骑手开发者本人mvp-主用户) 演进场景，[S4](../../product/personas.md#记录类)）需要为每辆车独立记录与统计油耗。

多车还是后续维保、提醒、多用户的**数据基座**——保养绑定到车、提醒按车触发、未来多用户的车归属用户。因此它是 Phase 2 的 **P0**。

## 2. 用户故事

- **US1** 作为有多辆车的骑手，我想登记多辆车（如"小绿""通勤车"），以便分别管理。
- **US2** 作为骑手，我想在记录时指定车辆（"小绿加了 10 升…"），以便记到正确的车上。
- **US3** 作为骑手，我想设一辆"当前默认车"，以便不指定时记到它，省去每次说车名。
- **US4** 作为骑手，我想按车查询统计（"小绿最近油耗"），以便分别看每辆车。
- **US5** 作为**现有单车用户**，我想在升级后历史数据自动归到一辆默认车，以便零损失平滑过渡。

## 3. 范围

**In Scope**
- 车辆的创建、列出、设为默认、按名称指代。
- 记录加油/里程时可选指定车辆，未指定用默认车。
- 统计查询可按车过滤。
- 存量数据迁移到自动创建的默认车。

**Out of Scope（本期不做）**
- 车辆删除/改名（可 Phase 2 后续补；删除涉及历史数据归属，谨慎）。
- 多用户（每辆车归属哪个用户）——Phase 3，但本期 schema **预留** `user_id` 演进位。
- 车辆详细档案（排量、年份、车架号等）——非油耗核心。

## 4. 验收标准（Given / When / Then）

- **AC1（登记）** Given 无车辆，When 用户说"添加一辆车 小绿"，Then 创建车辆"小绿"，若为首辆则自动设为默认。
- **AC2（默认记录）** Given 已有默认车"小绿"，When 用户说"加了 10 升花 98 里程 12580"（未指明车），Then 记录归属"小绿"。
- **AC3（指定记录）** Given 存在车辆"通勤车"，When 用户说"通勤车加了 5 升花 40 里程 8200"，Then 记录归属"通勤车"。
- **AC4（指代不明）** Given 存在"小绿""通勤车"两车且无明确默认，When 用户记录未指明车，Then Bot 反问"记到哪辆车？"。
- **AC5（按车统计）** Given 两车各有记录，When 用户问"小绿最近油耗"，Then 只统计"小绿"的记录。
- **AC6（列车 + 默认）** When 用户说"我有哪些车"，Then 列出所有车并标注默认车。
- **AC7（切默认）** Given 多车，When 用户说"默认车设成通勤车"，Then 后续未指明记录归属"通勤车"。
- **AC8（存量迁移）** Given 升级前已有加油记录，When 迁移执行，Then 自动创建一辆默认车且所有历史记录归属它，统计结果与升级前一致。

## 5. 交互示例

```
用户：添加一辆车，小绿
Bot：✅ 已添加车辆「小绿」，已设为默认车。

用户：小绿加了 10 升 95 号花 98，里程 12580
Bot：✅ 已记录（小绿）
     📍 里程：12,580 km
     ⛽ 加油：10 L × ¥9.80/L = ¥98.00
     📊 本次油耗：3.13 L/100km（距上次 319 km）

用户：我有哪些车
Bot：🏍 车辆列表
     • 小绿（默认）
     • 通勤车

用户：通勤车最近 3 次油耗
Bot：📊 通勤车 · 油耗统计
     …
```

## 6. 依赖与假设

- 依赖：现有 `fuel_records`/`mileage_records`、Agent 工具机制。
- 假设：单用户阶段，所有车归同一用户；车辆数量少（个位数），LLM 靠名称指代即可。

## 7. 开放问题

| 问题 | 影响 | 待决 |
|------|------|------|
| 车名重复/相似（"小绿"vs"小绿色"） | LLM 指代歧义 | 设计上靠精确名匹配 + 歧义反问（AC4 思路） |
| 车辆删除后历史记录如何处理 | 数据完整性 | 本期不做删除；未来用软删除（`is_active`） |
| 是否支持"当前里程"跨车混淆 | 误记到错车 | 默认车 + 指定车 + 歧义反问三层防护 |
