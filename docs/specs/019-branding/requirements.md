# 需求：品牌化 — 弼马温

> 规格 019 · 状态：📝 草稿 · 阶段：Phase 3 · 优先级：P2
> 关联：[design.md](design.md) · [tasks.md](tasks.md)

## 1. 问题陈述

项目原名"Moto Bot / Moto Agent / 摩托车油耗管家"缺乏品牌辨识度，名称不统一（不同位置用不同名字），用户感知散乱。需要一个统一、有记忆点的品牌名建立产品形象。

## 2. 用户故事

- 作为**终端用户**，我想看到一个统一、有温度的产品名称，而不是技术代号。
- 作为**开发者**，我想所有用户可见的品牌名称统一为同一标识，减少混淆。

## 3. 品牌名决策

**选定名：弼马温**

- 文化出处：孙悟空在天庭担任的官职"弼马温"，谐音"避马瘟"，本意是管马
- 产品关联：摩托车油耗管理 → "管车" → "弼马温"管马 → 幽默双关
- 优势：简洁（3 字）、有记忆点、有文化梗、无需 AI 后缀

## 4. 范围

**In Scope**
- Bot / PWA 所有用户可见的品牌名称统一替换
- 发件人名称、邮件标题
- 登录/绑定页面标题
- 系统提示词中的自我介绍
- 项目 README 和 CLAUDE.md 更新
- manifest.json（PWA 添加到主屏幕名称）

**Out of Scope（本期不做）**
- Logo / icon 重新设计（可后续）
- Worker 部署名（`wrangler.toml` `name`）— 改了就断部署 URL
- D1 数据库名（`moto-agent-db`）— 所有脚本依赖此名
- Docs 工程文档（不急，以后统一扫）
- 代码内部标识符（包名、变量名）— 不影响用户

## 5. 验收标准

- **AC1** Telegram Bot /start 回复中显示"弼马温"而非"Moto Bot/摩托车油耗管理助手"
- **AC2** Telegram /dashboard 链接文案显示"弼马温 Dashboard"
- **AC3** 登录邮件发件人显示"弼马温"，主题含"弼马温"
- **AC4** 浏览器登录/绑定确认页标题和 h1 显示"弼马温"
- **AC5** PWA 添加到主屏幕的名称为"弼马温"
- **AC6** PWA 页内标题显示"弼马温"
- **AC7** 系统提示词自我介绍改为"弼马温"
- **AC8** README.md 和 CLAUDE.md 标题更新为"弼马温"

## 6. 依赖与假设

- 无外部依赖
- SENDER_EMAIL 域名（@bbing.xyz）不变，只改发件人显示名
