# 需求：语音输入

> 规格 008 · 状态：✔️ 已实现（待部署 + 语音冒烟） · 阶段：Phase 2 · 优先级：P2
> 关联：[design.md](design.md) · [tasks.md](tasks.md) · [ADR-0007](../../engineering/adr/0007-cloudflare-workers-ai-stt.md)

## 1. 问题陈述

骑手在加油站戴手套、手不方便时，打字记录麻烦。希望直接发**语音**说一句"加了 10 升花 98 里程 12580"，系统转文字后照常记录（[S1 的语音变体](../../product/personas.md)）。

## 2. 用户故事

- **US1** 作为骑手，我想发一条语音说加油信息，系统转成文字并完成记录，无需打字。
- **US2** 作为骑手，我想在回复里看到"听到了什么"，以便发现识别错误并纠正（复用 spec 004 纠错）。
- **US3** 语音同样支持所有现有能力（查询、维保、提醒、改名…），因为转文字后走同一链路。

## 3. 范围

**In Scope**
- 接收 Telegram 语音消息（`message:voice`，OGG/Opus）。
- 用 Cloudflare Workers AI（Whisper）转写为中文文本。
- 回显识别文本（"🎙 听到：…"），再走现有 `runAgent`。
- 时长上限与失败兜底。

**Out of Scope（本期不做）**
- 语音回复（TTS）——只做语音输入，回复仍文字。
- 音频文件/视频消息（`message:audio`/`video_note`）——仅 `voice`。
- 多语言——本期按中文识别（`language=zh`）。
- 转写置信度可视化/逐词高亮。

## 4. 验收标准（Given / When / Then）

- **AC1** Given 白名单用户发送语音"加了 10 升花 98 里程 12580"，When bot 处理，Then 先回显"🎙 听到：…"，再按文本完成 `log_fuel` 记录与油耗回复。
- **AC2** Given 语音内容是查询/维保/提醒等，When 处理，Then 转文字后走对应工具（与打字一致）。
- **AC3（识别为空）** Given 语音无法识别出文字，When 处理，Then 提示"没听清，请再说一遍或直接打字"。
- **AC4（识别失败）** Given STT 调用异常，When 处理，Then 提示失败并建议打字，不崩。
- **AC5（时长上限）** Given 语音超过上限（如 60s），When 处理，Then 提示过长、请精简或打字，不调用 STT。
- **AC6（纠错闭环）** Given 识别把里程听错，When 用户看到回显发现错误，Then 可用"上一条里程改成 X"（spec 004）纠正。

## 5. 交互示例

```
用户：[语音] 加了10升95号花98里程12580
Bot：🎙 听到：加了10升95号花98里程12580
Bot：✅ 已记录（小绿）
     📍 里程：12,580 km
     ⛽ 10 L × ¥9.80/L = ¥98
     📊 本次油耗：3.13 L/100km（距上次 319 km）
```

## 6. 依赖与假设

- 依赖：Cloudflare Workers AI（`[ai]` binding，账户启用，[ADR-0007](../../engineering/adr/0007-cloudflare-workers-ai-stt.md)）。
- 假设：语音以中文为主；音频短（加油记录通常 < 15s）；Whisper 直接接受 OGG/Opus，无需转码。

## 7. 开放问题

| 问题 | 影响 | 待决 |
|------|------|------|
| 中文数字识别（"三万四千六" vs 34600） | 解析准确率 | 回显让用户可发现；LLM 解析两种写法都支持；必要时切 OpenAI（ADR-0007 可替换） |
| Workers AI 配额/延迟 | 可用性 | 监控 `[voice]` 日志；失败兜底提示打字 |
