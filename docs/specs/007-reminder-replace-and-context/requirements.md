# 需求：提醒去重 + 会话回合截断（修复）

> 规格 007 · 状态：✔️ 已实现 · 阶段：Phase 2（修复） · 优先级：P1
> 关联：[design.md](design.md) · [tasks.md](tasks.md) · 修复 [spec 003](../003-reminders/) / [spec 006](../006-hardening/) 暴露的两个缺口

上线后审查发现的两个真实缺口，合并为一个修复 spec。

## A. 提醒去重（改间隔会叠加）

### 问题
`set_reminder` 只新增、不替换。用户先设"机油每 3000 公里"，再说"改成每 2000"，会**叠加成两条** active 提醒（3000 与 2000），各自独立触发与续期 → 收到重复提醒。

### AC
- **AC-A1** Given 已有"小绿·机油"提醒，When 再设"小绿·机油"提醒，Then 旧的作废、只保留新的（不叠加），回复提示"已更新"。
- **AC-A2** 替换按 **车 + 类型** 精确隔离：不影响同车其它类型、不影响其它车同类型。

## B. 会话历史按回合截断（可能切坏 tool 配对）

### 问题
历史截断用 `slice(-10)` 机械按条数切，可能切在「assistant 的 tool_calls」与其「tool 结果」之间，导致下次请求出现悬空配对，被模型 API 拒绝。

### AC
- **AC-B1** 截断后的历史**必从 `user` 消息开始**，不出现悬空的 tool 结果 / assistant(tool_calls) 在开头。
- **AC-B2** 在 ≤ maxMessages 的前提下尽量多保留完整回合；单个回合超长时，宁可超额也从该回合的 `user` 起，保证配对完整。
- **AC-B3** 短历史 / 空历史行为不变。

## 范围
**In Scope**：set_reminder 替换；会话历史回合对齐截断。
**Out of Scope**：MAX_ROUNDS 超时导致历史以 tool 结果结尾的罕见情形（不在本次处理）；上下文压缩/摘要（未来）。
